- 支持自定义技术指标和回测策略

## 3. 踩坑记录

### 3.1 数据获取

#### 问题1: Tushare API 限流
- **现象**: `每分钟最多访问该接口1次`
- **原因**: Token 权限限制
- **解决**: 等待限流恢复后继续获取

#### 问题2: 股票代码转换
- **现象**: `stock_basic` 返回纯数字代码如 `000001`
- **解决**: 手动转换 `0`开头 → `SZ`, `6`开头 → `SH`

#### 问题3: 部分股票缺失
- **现象**: 4587只股票中5只未获取
- **原因**: 获取过程中偶发失败
- **解决**: 手动补采缺失股票

### 3.2 数据库导入

#### 问题1: UNIQUE constraint failed
- **现象**: 重复插入报错
- **原因**: 已有部分数据在表中
- **解决**: 使用 `if_exists='append'` 跳过重复

### 3.3 前端部署 (Streamlit Cloud)

#### 问题1: Plotly 子图行列不匹配
- **错误**: `The row and col arguments are invalid`
- **原因**: 动态计算的 `row_count` 与实际添加的 trace 行数不一致
- **解决**: 固定3行子图，用隐藏标题控制显示

#### 问题2: `updateannotations` 数组越界
- **错误**: `list index out of range`
- **原因**: 只有2个子图时访问第3个
- **解决**: 添加长度检查 `len(fig.layout.annotations) > n`

#### 问题3: 线条显示为虚线
- **错误**: 所有均线、指标线显示为断开的虚线
- **原因**: 数据中有 NaN 值
- **解决**: 添加 `connectgaps=True` 参数 ⚠️
- **经验**: 所有 `go.Scatter` 必须加 `connectgaps=True`

#### 问题4: `connectgaps` 参数位置错误
- **错误**: Plotly 报错 `connectgaps` 不是 line 的属性
- **错误写法**: `line=dict(connectgaps=True, color='...')`
- **正确写法**: `go.Scatter(..., connectgaps=True, line=dict(...))`
- **经验**: `connectgaps` 是 trace 级别参数，不是 line 级别

#### 问题5: Streamlit `st.columns()` 返回容器问题
- **错误**: `st.columns(...).checkbox()` 报错
- **原因**: `st.columns()` 返回的容器不支持直接调用 `checkbox()`
- **解决**: 直接使用 `st.checkbox()` 而非嵌套在 columns 中
